最重要的是理解了git的工作原理
知道了工作区就是就是我们开发的区域。
.git 里面包含了两个东西，一个是暂存区，一个版本库。
git add . 是把修改添加暂存区。
git commit 是把修改提交到版本库。
分支就是指向最新版本的一个指针，而当前是在哪一个分支，是通过Head指针来指向。这也就是git提交代码那么远的原型，只是保存了修改，只需要把分支指针往后移动就行了。还有切换分支就是把Head指针指向另外一个分支就好了。

还有多人合作，先git pull拉取，如果多人同时修改了一个文件，就可能会产生冲突。我们得先解决冲突，然后提交一个版本，最后通过git push 到远程。

反正，git的确比svn来得更加得心应手。
